Project Description

This app is a trading-card index-fund prototype (Pokemon, Magic: The Gathering, Yu-Gi-Oh!). It calculates a live index price by combining card prices proportionally to each card’s basket contribution, then lets you explore history, basket composition, and paper-trade the index funds.

How Pricing Works

Index Price = Σ(card price × basket contribution % / 100)

With the Node backend running, prices come from the TCG API when TCGAPI_KEY is set. The first load picks a basket via search, then locks card IDs and each row’s weight %. Refresh only updates £ prices from the API; weight % columns stay fixed until you restart the server. Index NAV and history use those frozen weights. Prices are rounded to 2p to reduce noise.

What This Solves

Most card investing is concentrated and operationally heavy. You need to choose individual cards, manage storage, and accept single-card risk if one position underperforms. This project shifts that model to index-style exposure so users track a weighted basket instead of one collectible.

Each index fund targets one major TCG: Pokemon, Magic: The Gathering, or Yu-Gi-Oh!, using search once to build a basket, then stable card IDs with live price updates on refresh.

How To Use The App

1. Open Index Marketplace to compare index funds. 2. Select a fund to inspect history and constituents. 3. Use Basket Builder to see set-level contribution and weighted value. 4. Use Paper Trading to simulate buy and sell actions. 5. Use Data Cache to validate cache and database status.

Project Overview

This app is a multi-game TCG index-fund prototype. It tracks market movement by aggregating card prices into weighted baskets (one per game) and exposing each basket as a single index price.

How The Index Works

Index Price = sum(card price * basket contribution % / 100)

Each card contributes according to its basket weight. This gives diversified exposure across multiple cards and sets, rather than relying on one collectible.
